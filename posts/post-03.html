Electron 앱의 메모리 누수 디버깅: Chrome DevTools 실전 절차

게시일: 2026년 1월 28일 · 15분 읽기

Electron 앱은 Main/Renderer/Preload가 얽혀 있어 누수 지점을 놓치기 쉽습니다. 가장 빠른 접근은 증상 재현 경로를 고정하고, 힙 스냅샷을 세 번 이상 비교해 증가 객체를 특정하는 것입니다.

1단계: 재현 시나리오 고정

사용자 보고를 그대로 따라가기보다, 메모리 증가가 분명한 최소 동작 집합을 정의합니다. 예: 검색 열기 → 상세 이동 → 뒤로가기 20회 반복. 시나리오를 스크립트화하면 릴리즈마다 회귀 검증이 쉬워집니다.

2단계: Renderer 힙 스냅샷 비교

DevTools에서 baseline/suspect/post-gc 스냅샷을 비교해 Detached DOM tree, Listener, Closure retaining path를 우선 확인합니다. React 프로젝트라면 stale subscription과 캐시 레이어의 강한 참조가 자주 원인입니다.

3단계: Main 프로세스 점검

IPC 핸들러 등록 해제가 누락되면 Main 메모리가 서서히 증가합니다. 이벤트 바인딩은 등록과 해제를 같은 모듈에서 관리하고, 창 생명주기와 함께 dispose 되도록 통일하는 것이 안전합니다.

운영 적용

메모리 예산(예: Renderer 300MB)을 명시하고 경고 임계치를 넘으면 로그와 힙 덤프를 자동 수집하도록 구성했습니다. 그 결과 원인 파악 시간이 크게 줄었고, 장시간 실행 세션 안정성이 개선됐습니다.

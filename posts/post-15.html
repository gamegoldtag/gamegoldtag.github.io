바이브 코딩으로 10분만에 무료 웹사이트 만들기

게시일: 2025년 11월 8일 · 13분 읽기

바이브 코딩(Vibe Coding)은 "코드를 직접 작성"하기보다 "원하는 결과를 설명하고 AI와 반복 수정"하는 작업 방식입니다. 짧은 시간에 초안을 만드는 데 특히 강하고, 소개 페이지나 랜딩 페이지 같은 정적 웹사이트 제작에 적합합니다.

바이브 코딩이란?

Vibe Coding은 분위기(vibe)와 목적을 설명하면 AI가 HTML/CSS/JS 초안을 생성해 주는 방식입니다. 기존 개발 흐름보다 시작 장벽이 낮지만, 공개 전에는 사용자 검수와 기본 테스트가 필요합니다.

기존 방식: 아이디어 → 설계 → 코딩 → 테스트 → 배포
바이브 코딩: 아이디어 → AI 설명 → 결과 확인 → 수정 → 배포

코드를 직접 이해하고 작성하는 것과는 다릅니다. 바이브 코딩에서 사람의 역할은 "코드를 쓰는 것"이 아니라 "무엇을 만들지 정확히 전달하고, 결과를 판단하는 것"입니다.

정말 10분이면 되나요?

초안 기준으로는 가능합니다. 다만 초안은 공개 가능한 최소 버전이며, 이미지 교체·세부 수정·다중 페이지 구성은 추가 시간이 필요합니다.

  • 10분 내 가능: 소개 페이지, 디지털 명함, 이벤트 안내, 단일 랜딩 페이지
  • 30분~1시간: 실제 이미지 반영, 고급 애니메이션, 복수 페이지 내비게이션
  • 하루 이상: 다국어 지원, 블로그 기능, 관리자 패널

10분 실전 타임라인

0:00 - 목적 정하기

무엇을 위한 사이트인지 한 문장으로 정합니다. 예: "카페 홍보용 단일 페이지 사이트"

1:00 - 프롬프트 작성 및 제출

ChatGPT 또는 Claude를 열고 아래 형식으로 요청합니다. 구체적일수록 수정 횟수가 줄어듭니다:

용도: 소개 페이지
분위기: 미니멀, 깔끔함
색상: 파란색 포인트 (#2196F3), 흰 배경
구성: 상단 소개(슬로건 + 설명) - 특징 3가지(아이콘+텍스트) - 연락처
요구사항:
  - 모바일 반응형
  - 한 파일(HTML/CSS/JS 모두 포함)
  - Pretendard 폰트(CDN)
  - 언어: 한국어

2:00~4:00 - 코드 저장 및 첫 확인

생성된 코드를 복사해 index.html로 저장하고 브라우저로 엽니다.

  • Windows: 메모장 → "파일 형식: 모든 파일" → index.html로 저장
  • macOS: TextEdit → "포맷" → "일반 텍스트 만들기" → 저장

화면이 예상과 다른 부분을 메모합니다.

5:00~8:00 - 수정 2~3 라운드

짧고 명확한 지시를 반복합니다. 한 번에 여러 가지를 바꾸면 코드가 섞일 수 있으므로 2~3가지씩 묶어서 요청합니다:

라운드 1:
- 상단 배경을 더 어둡게 (#1565C0)
- 버튼에 hover 시 색상 변화 효과 추가

라운드 2:
- 특징 카드에 shadow 효과 추가
- 하단 연락처 섹션 패딩 증가

라운드 3:
- 전체 폰트 크기 약간 키우기
- 모바일에서 카드 1열로 표시

9:00~10:00 - 무료 배포

Netlify, GitHub Pages, Cloudflare Pages 중 하나로 배포합니다. 초보자는 Netlify 드래그앤드롭 배포가 가장 쉽습니다.

  1. netlify.com 접속 → 가입
  2. "Add new site" → "Deploy manually"
  3. 폴더 드래그앤드롭 → 배포 완료
  4. *.netlify.app 주소 발급 확인

초안 이후 품질 올리는 방법

10분 초안은 구조만 잡힌 버전입니다. 아래 순서대로 개선하면 실사용 품질로 완성됩니다:

이미지 교체

AI가 만든 코드에는 unsplash 같은 예시 이미지 URL이 들어가는 경우가 많습니다. 실제 사진이나 라이선스 확인된 이미지로 교체하세요.

  • Unsplash (unsplash.com): 고화질 무료 이미지, 상업 사용 가능
  • Pexels (pexels.com): 무료 이미지/영상
  • 직접 촬영한 사진이 가장 효과적

한글 폰트 적용

Pretendard는 한글 웹폰트 중 가독성이 좋고 CDN으로 쉽게 적용할 수 있습니다:

<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/orioncactus/pretendard/dist/web/variable/pretendardvariable.min.css">
<style>
  body { font-family: "Pretendard Variable", "Pretendard", sans-serif; }
</style>

SEO 기본 설정

사이트가 검색엔진에 노출되려면 head 섹션에 기본 메타 태그가 있어야 합니다:

<title>홍길동 | 프리랜서 그래픽 디자이너</title>
<meta name="description" content="브랜드 아이덴티티를 만드는 그래픽 디자이너">
<meta property="og:title" content="홍길동 포트폴리오">
<meta property="og:description" content="로고 디자인, 브랜딩, SNS 콘텐츠">

커스텀 도메인 연결

브랜드 신뢰도를 높이려면 커스텀 도메인을 연결합니다. 국내에서는 가비아, 후이즈 등에서 연간 1~3만 원대로 도메인을 구입할 수 있습니다.

바이브 코딩이 막히는 상황과 해결법

  • 코드가 너무 길어 복사 실패: "전체 코드가 아니라 수정된 부분만 알려주세요"라고 요청
  • 모바일 레이아웃이 계속 깨짐: "모바일 반응형을 추가해주세요. viewport meta 태그 포함해서"라고 구체적으로 재요청
  • 이전 수정이 사라짐: 매 라운드마다 전체 코드를 저장하는 습관을 만들어두기
  • 원하는 디자인이 안 나옴: 참고 사이트를 설명하거나, 색상 코드(#RRGGBB)와 구체적인 레이아웃 명시

주의할 점

  • AI 결과물은 사실·법적 검토를 대신하지 않습니다.
  • 저작권이 불명확한 이미지·문구는 그대로 사용하지 않는 것이 안전합니다.
  • 호스팅 무료 정책과 한도는 시점별로 변경될 수 있습니다.
  • 결제·개인정보를 처리하는 기능은 AI 코드만으로 안전하게 구현하기 어렵습니다.
핵심은 "10분 만에 완성품"이 아니라 "10분 만에 공개 가능한 초안"을 만든 뒤, 검수와 개선으로 완성도를 끌어올리는 방식입니다.

마무리

바이브 코딩은 웹사이트 제작의 시작 비용과 진입 장벽을 크게 낮춰줍니다. 코딩 경험이 없어도 결과를 빠르게 확인할 수 있고, 그 결과를 기준으로 개선 방향을 잡을 수 있다는 점이 가장 큰 장점입니다. 처음엔 어색하지만, 2~3번 반복하면 AI와 함께 일하는 리듬이 생깁니다. 그때부터는 아이디어가 생기는 즉시 실제 웹 페이지로 옮기는 것이 자연스러워집니다.

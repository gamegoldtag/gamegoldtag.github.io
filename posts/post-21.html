남는 안드로이드 폰으로 웹서버 만들기 (Termux)

게시일: 2025년 9월 26일 · 14분 읽기

서랍 속 구형 안드로이드 폰도 테스트 서버로 활용할 수 있습니다. Termux를 사용하면 리눅스 명령어를 실행할 수 있고, 정적 파일 서버 또는 Node 기반 서버를 쉽게 띄울 수 있습니다. 비용을 거의 들이지 않고 서버·네트워크 기본 개념을 실습하기에도 좋은 방식입니다.

이 방식이 적합한 용도

  • 개인 테스트 서버 (개발 중 기능 확인)
  • 사내 Wi-Fi 내부망 데모 서버
  • 임시 파일 공유 서버
  • 서버·Linux 명령어 학습 환경

24시간 상용 서비스 운영보다는 테스트·데모·학습 목적에 적합합니다.

준비물

  • 안드로이드 7.0 이상 기기
  • 지속 전원 연결 (충전 케이블 상시 연결)
  • Wi-Fi 연결
  • 여유 저장공간 (최소 500MB 이상)

1단계: Termux 설치

Google Play Store의 Termux는 더 이상 업데이트가 되지 않습니다. 최신 버전을 유지하려면 F-Droid를 통한 설치를 권장합니다.

1) 브라우저에서 f-droid.org 접속
2) "F-Droid" 앱 다운로드 및 설치 (설치 전 "알 수 없는 앱 허용" 설정 필요)
3) F-Droid 앱 실행 후 "termux" 검색
4) Termux 설치

2단계: 초기 설정

Termux를 처음 실행한 후 패키지를 업데이트하고 저장소 접근 권한을 설정합니다:

pkg update && pkg upgrade -y
termux-setup-storage

termux-setup-storage를 실행하면 권한 요청 팝업이 뜹니다. "허용"을 선택해야 폰의 저장소 파일에 접근할 수 있습니다.

3단계: 서버 패키지 설치

용도에 따라 두 가지 옵션이 있습니다:

lighttpd (정적 파일 서버)

HTML/CSS/JS 파일만 서빙하는 가벼운 서버입니다:

pkg install lighttpd -y

Node.js (유연한 서버)

동적 API 응답이나 서버 로직이 필요한 경우:

pkg install nodejs -y
node -v  # 버전 확인

4단계: 서버 실행

lighttpd로 정적 서버 실행

mkdir -p ~/www
cd ~/www
echo "<h1>내 폰 웹서버</h1>" > index.html
lighttpd -D -f /data/data/com.termux/files/usr/etc/lighttpd/lighttpd.conf

Node.js로 정적 서버 실행 (npx serve)

mkdir -p ~/www
cd ~/www
echo "<h1>내 폰 웹서버</h1>" > index.html
npx serve . -p 8080

Node.js Express 서버

mkdir -p ~/myapp && cd ~/myapp
npm init -y
npm install express

# server.js 파일 생성
cat > server.js <<'EOF'
const express = require('express');
const app = express();
app.use(express.static('public'));
app.get('/api/status', (req, res) => {
  res.json({ status: 'ok', time: new Date() });
});
app.listen(8080, () => console.log('서버 실행: http://localhost:8080'));
EOF

node server.js

5단계: 같은 Wi-Fi에서 접속 확인

폰의 IP 주소를 확인합니다:

ifconfig | grep "inet "
# 또는
ip addr show wlan0 | grep "inet "

동일 Wi-Fi에 연결된 PC나 태블릿에서 http://192.168.0.15:8080 형태로 접속해 서버가 동작하는지 확인합니다.

6단계: 배터리 최적화 해제

안드로이드는 배터리 절약을 위해 백그라운드 앱을 제한합니다. Termux가 계속 실행되려면 배터리 최적화를 해제해야 합니다:

  • 설정 → 배터리 → 배터리 최적화
  • 앱 목록에서 "Termux" 찾기
  • "최적화 안 함" 선택

Termux 앱 내에서도 wake lock을 설정할 수 있습니다:

termux-wake-lock

외부 공개: ngrok 터널링

같은 Wi-Fi 외부에서도 접속하게 하려면 터널링 도구를 사용합니다. ngrok이 가장 널리 쓰입니다:

# ngrok 다운로드 (ARM64 버전)
pkg install wget -y
wget https://bin.equinox.io/c/bNyj1mQVY4c/ngrok-v3-stable-linux-arm64.tgz
tar -xzf ngrok-v3-stable-linux-arm64.tgz

# ngrok.com 가입 후 인증 토큰 설정
./ngrok config add-authtoken YOUR_TOKEN

# 터널 열기
./ngrok http 8080

터널이 열리면 https://xxxx.ngrok-free.app 형식의 주소가 생성됩니다. 이 주소를 다른 사람과 공유하면 외부에서도 접속이 가능합니다.

ngrok 무료 플랜의 주소는 세션 재시작 시 변경됩니다. 고정 주소가 필요하면 유료 플랜 또는 Cloudflare Tunnel을 검토하세요.

현실적인 한계와 주의점

  • 발열 관리: 장시간 서버 실행 시 발열이 발생합니다. 통풍이 잘 되는 곳에 놓거나 케이스를 제거하세요.
  • 배터리 수명: 항상 충전 상태로 운영하면 배터리가 빨리 노후화됩니다. 80% 수준을 유지하는 것이 좋습니다.
  • 네트워크 안정성: 가정용 Wi-Fi 환경에 따라 서비스 안정성이 달라집니다.
  • 보안: 외부에 공개되는 서버라면 불필요한 포트 노출을 최소화하고, ngrok 토큰을 타인에게 공유하지 마세요.
  • 터널 주소 변경: ngrok 무료 플랜의 주소는 재시작 시 변경됩니다.

마무리

구형 폰을 재활용해 테스트 서버를 만들면 로컬 검증과 임시 공유가 훨씬 쉬워집니다. 비용을 거의 들이지 않고 네트워크, 서버 기본 개념, Linux 명령어를 실습하기에도 좋은 환경입니다. 24시간 상용 서비스보다는 개발 중 기능 확인이나 팀 내 데모 공유 목적에서 가장 효율적으로 활용할 수 있습니다.

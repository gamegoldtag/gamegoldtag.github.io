남는 안드로이드 폰으로 웹서버 만들기 (Termux)

게시일: 2025년 9월 26일 · 12분 읽기

서랍 속 구형 안드로이드 폰도 테스트 서버로 활용할 수 있습니다. Termux를 사용하면 리눅스 명령어를 실행할 수 있고, 정적 파일 서버 또는 Node 기반 서버를 쉽게 띄울 수 있습니다.

이 방식이 적합한 용도

  • 개인 테스트 서버
  • 개발 중인 화면 임시 공유
  • 내부망 데모 서버

준비물

  • 안드로이드 7.0 이상 기기
  • 지속 전원(충전기)
  • Wi-Fi 연결
  • 여유 저장공간

1단계: Termux 설치

최신 유지 버전을 위해 F-Droid 경로 설치를 권장합니다.

1) f-droid.org 접속
2) F-Droid 설치
3) F-Droid에서 termux 검색 후 설치

2단계: 초기 설정

pkg update && pkg upgrade -y
termux-setup-storage

3단계: 서버 패키지 설치

정적 파일만 서빙하면 lighttpd, 유연한 구성이 필요하면 Node.js를 사용합니다.

pkg install lighttpd -y
# 또는
pkg install nodejs -y

4단계: 서버 실행

lighttpd

mkdir -p ~/www
cd ~/www
echo "<h1>내 폰 웹서버</h1>" > index.html
lighttpd -D -f /data/data/com.termux/files/usr/etc/lighttpd/lighttpd.conf

Node.js(serve)

mkdir -p ~/www
cd ~/www
echo "<h1>내 폰 웹서버</h1>" > index.html
npx serve .

5단계: 같은 Wi-Fi에서 접속

ifconfig | grep inet
# 예: 192.168.0.15

동일 네트워크 PC에서 http://192.168.0.15:8080 형태로 접속해 확인합니다.

외부 접속(선택): ngrok

외부 공유가 필요하면 터널링 도구를 사용합니다.

pkg install wget -y
wget https://bin.equinox.io/c/bNyj1mQVY4c/ngrok-v3-stable-linux-arm64.tgz
tar -xzf ngrok-v3-stable-linux-arm64.tgz
./ngrok config add-authtoken YOUR_TOKEN
./ngrok http 8080

현실적인 한계

  • 상시 전원/발열 관리 필요
  • 가정용 네트워크 안정성에 영향 받음
  • 무료 터널 주소는 변동될 수 있음
  • 배터리 수명 단축 가능성
이 방식은 24시간 상용 서비스보다는 테스트/데모 환경에 적합합니다. 공개 서비스 운영은 관리형 호스팅 또는 클라우드 인프라를 권장합니다.

마무리

구형 폰을 재활용해 테스트 서버를 만들면 로컬 검증과 공유가 훨씬 쉬워집니다. 비용을 거의 들이지 않고 네트워크/서버 기본 개념을 실습하기에도 좋은 방식입니다.

AI가 만들어주는 나만의 무료 웹사이트

게시일: 2025년 11월 22일 · 14분 읽기

"내 웹사이트 하나 갖고 싶은데 개발은 모르고, 돈도 부담된다"는 고민이 있습니다. 2025년 기준으로는 AI 도구를 활용해 초기 비용 없이 개인 사이트를 시작할 수 있습니다. 이 글에서는 AI로 웹사이트를 만들고 무료 호스팅에 올리는 전체 흐름을 단계별로 정리합니다.

핵심은 한 번에 완성품을 받으려는 기대를 내려놓는 것입니다. AI는 초안을 빠르게 만들어주는 도구이고, 사람이 방향을 잡고 반복 수정하는 과정이 최종 품질을 결정합니다.

AI가 코드를 짜준다는 말, 어디까지 맞을까?

ChatGPT, Claude 같은 도구에 원하는 화면과 기능을 구체적으로 설명하면 HTML/CSS/JS 초안을 빠르게 만들 수 있습니다. 다만 결과물은 "완성본"이 아니라 "초안"으로 보는 게 정확합니다. 실제 공개 전에는 내용 검수, 오탈자 점검, 모바일 확인이 꼭 필요합니다.

AI가 잘하는 것과 못하는 것을 구분하면 활용 방향이 명확해집니다:

AI가 잘하는 것사람이 해야 하는 것
HTML/CSS/JS 구조 생성내용 검증 및 오탈자 수정
레이아웃 반복 수정실제 이미지/정보 교체
색상·폰트 조정모바일 실기기 확인
애니메이션·효과 추가저작권·개인정보 검토

전체 흐름 한눈에 보기

  1. 만들고 싶은 사이트 목적과 구성 정리
  2. AI에게 구체적인 프롬프트로 코드 요청
  3. 코드 복사 후 index.html로 저장
  4. 브라우저에서 확인하며 수정 반복
  5. 무료 호스팅에 업로드
  6. 도메인·메타데이터 정리 후 공개

1단계: 사이트 구성 먼저 정리하기

AI에게 요청하기 전, 무엇을 만들지 텍스트로 정리하는 것이 중요합니다. 막연한 요청은 막연한 결과를 낳습니다. 아래 항목들을 미리 정해두면 프롬프트 품질이 크게 올라갑니다:

  • 사이트 목적: 포트폴리오? 업체 소개? 이벤트 안내?
  • 주요 섹션: 상단 소개, 서비스 목록, 작업물, 연락처 등
  • 디자인 분위기: 미니멀, 화려함, 전문적, 친근함
  • 색상: 메인 컬러와 배경 색상
  • 기술 요건: 모바일 반응형, 한 파일 또는 분리 구성

2단계: AI에게 구체적으로 요청하기

좋은 프롬프트일수록 수정 횟수가 줄어듭니다. 아래처럼 조건을 명시해서 요청하세요:

효과적인 프롬프트 예시

포트폴리오 사이트를 만들어주세요.
- 섹션 구성: 상단 자기소개 → 기술 스택 카드 4개 → 프로젝트 3개 → 연락처
- 디자인: 다크 테마, 포인트 컬러 #3498db
- 조건: 모바일 반응형, HTML/CSS/JS 한 파일로
- 폰트: Pretendard (CDN 링크 사용)
- 언어: 한국어

아쉬운 프롬프트 예시

웹사이트 만들어줘        (X - 너무 막연함)
예쁜 포트폴리오 만들어줘 (X - 조건 불명확)

3단계: 코드 저장 후 로컬 확인

생성된 코드를 복사해 index.html로 저장하고 브라우저에서 직접 확인합니다.

운영체제별 저장 방법

  • Windows: 메모장 열기 → 붙여넣기 → 파일 저장 시 "파일 형식: 모든 파일" 선택 → index.html로 저장 (*.txt 확장자 방지)
  • macOS: TextEdit 열기 → 포맷 메뉴에서 "일반 텍스트 만들기" 선택 → 붙여넣기 → index.html로 저장
  • VS Code 추천: 설치 후 새 파일 → index.html로 저장 → 코드 붙여넣기 (미리보기 기능 활용 가능)

저장 후 파일을 더블클릭하면 기본 브라우저로 열립니다. 화면을 보며 수정할 부분을 목록으로 적어두세요.

4단계: 작은 단위로 수정 반복하기

한 번에 모든 것을 바꾸려 하면 오히려 시간이 더 걸립니다. 한 번 요청에 2~3가지 변경 사항을 묶어서 요청하는 방식이 효율적입니다:

아래 3가지만 수정해주세요:
1) 상단 제목 폰트 크기를 더 크게
2) 서비스 섹션 배경을 연한 회색으로 변경
3) 하단에 이메일 연락 버튼 추가

수정된 코드를 다시 저장하고 새로고침하며 확인합니다. 이 과정을 3~5회 반복하면 초안에서 완성도 높은 사이트로 빠르게 발전합니다.

5단계: 무료 호스팅에 배포하기

서비스특징난이도추천 대상
Netlify드래그앤드롭 배포, 쉬운 UI낮음초보자, 빠른 시작
GitHub Pages버전 관리, Git 연동중간개발자, 코드 이력 필요
Cloudflare Pages글로벌 CDN, 빠른 로딩중간성능 중시, 글로벌 트래픽

Netlify 배포 5단계 (초보자 추천)

  1. netlify.com에서 무료 계정 가입
  2. 대시보드에서 "Add new site" 선택
  3. "Deploy manually" 클릭
  4. index.html이 있는 폴더를 드래그앤드롭
  5. 배포 완료 후 *.netlify.app 주소 발급 확인

GitHub Pages 배포 5단계

  1. GitHub에서 새 저장소 생성 (예: my-portfolio)
  2. index.html 업로드 및 커밋
  3. Settings → Pages에서 배포 브랜치(main) 선택
  4. 몇 분 후 username.github.io/my-portfolio URL 활성화
  5. 모바일·데스크톱 최종 점검 후 공개

어떤 사이트에 적합한가?

AI + 무료 호스팅 조합은 다음 유형에서 충분히 실용적으로 동작합니다:

  • 개인 포트폴리오, 소개 페이지, 이력서 페이지
  • 가게·업체 안내, 메뉴/가격표, 행사 안내 페이지
  • 프로젝트 데모, 기술 문서, 오픈소스 소개 페이지
  • 디지털 명함, 링크 모음 페이지

배포 후 SEO 기초 챙기기

사이트를 공개했다면 검색엔진에 잘 노출되도록 기본적인 설정을 챙기는 것이 좋습니다. AI에게 아래 항목을 추가해달라고 요청하면 됩니다:

<!-- head 섹션에 추가 -->
<meta name="description" content="사이트 설명 (100-150자)">
<meta property="og:title" content="페이지 제목">
<meta property="og:description" content="SNS 공유 시 설명">
<title>페이지 제목 | 이름</title>

Google Search Console에 사이트를 등록하면 검색 노출 현황을 무료로 확인할 수 있습니다. 등록 후 사이트맵을 제출하면 색인 속도가 빨라집니다.

자주 막히는 상황과 해결법

  • AI 코드가 브라우저에서 이상하게 보임: 코드를 전체 복사했는지, <!DOCTYPE html>로 시작하는지 확인
  • 이미지가 안 보임: 이미지 파일을 index.html과 같은 폴더에 넣고, 파일명 대소문자 확인
  • 모바일에서 레이아웃 깨짐: AI에게 "모바일 반응형"을 명시해서 다시 요청
  • 배포 후 수정 반영 안 됨: 브라우저 강력 새로고침(Ctrl+Shift+R) 또는 새 배포 실행

한계와 주의사항

  • 로그인·결제 같은 복잡한 기능은 별도 백엔드가 필요합니다.
  • AI 생성 코드에는 버그가 있을 수 있으니 다양한 기기에서 테스트가 필수입니다.
  • 이미지·문구의 저작권과 개인정보 포함 여부를 반드시 확인하세요.
  • 무료 플랜의 한도와 정책은 시점에 따라 변경될 수 있습니다.
요금제, 무료 한도, 서비스 정책은 시점에 따라 변경될 수 있으므로 배포 전 공식 문서를 확인하는 것이 안전합니다.

마무리

핵심은 "완벽한 코드를 한 번에 받기"가 아니라 "AI 초안을 빠르게 만들고 사람이 방향을 잡아 완성하는 것"입니다. 이 흐름만 익히면 비개발자도 수월하게 개인 웹사이트를 운영할 수 있습니다. 처음에는 시간이 걸리지만, 두 번째 사이트부터는 훨씬 빠르게 만들 수 있게 됩니다.

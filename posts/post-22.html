구형 노트북을 홈서버로 - ngrok으로 외부 공개까지

게시일: 2025년 9월 19일 · 13분 읽기

구형 노트북은 홈서버 용도로 재활용 가치가 높습니다. 폰 기반 서버보다 발열과 성능 여유 측면에서 유리하고, 키보드/화면이 있어 관리도 편합니다. 이 글에서는 Node.js 서버를 띄우고 ngrok으로 외부 공개하는 흐름을 정리합니다.

노트북 홈서버가 유리한 이유

  • 장시간 구동 시 발열/전원 관리가 상대적으로 안정적
  • 성능 여유가 있어 웹서버 + 간단한 API 동시 운영 가능
  • 입출력 장치가 있어 장애 대응이 쉬움

준비물

  • 구형 노트북(Windows/macOS/Linux)
  • 지속 전원 연결
  • 인터넷 연결
  • ngrok 계정(외부 공개용)

1단계: Node.js 설치

LTS 버전을 설치한 뒤 버전을 확인합니다.

node -v
npm -v

2단계: 로컬 서버 실행

정적 파일 서버

npx serve .

Express 서버

mkdir my-server && cd my-server
npm init -y
npm install express
// server.js
const express = require('express');
const app = express();

app.use(express.static('public'));
app.get('/api/hello', (req, res) => res.json({ message: '홈서버 작동 중!' }));

app.listen(3000, () => console.log('http://localhost:3000'));
node server.js

3단계: ngrok으로 외부 공개

ngrok 설치 후 토큰을 등록하고 터널을 엽니다.

ngrok config add-authtoken YOUR_TOKEN
ngrok http 3000

출력되는 https://*.ngrok-free.app 주소로 외부 접속이 가능합니다.

4단계: 절전/덮개 설정

노트북이 절전 모드로 전환되면 서버가 멈추므로 덮개 동작과 절전 정책을 조정합니다. 운영체제 전원 설정에서 "덮개 닫을 때 동작"과 "절전 진입"을 확인하세요.

서버 자동 시작(pm2)

npm install -g pm2
pm2 start server.js --name my-server
pm2 startup
pm2 save

재부팅 후에도 서버 프로세스를 자동 복구할 수 있습니다.

현실적인 한계

  • 가정용 인터넷 장애 시 서비스 중단
  • ngrok 무료 주소는 재시작 시 변경 가능
  • 전기 사용량 및 하드웨어 노후화 고려 필요
  • 포트/접근 제어 등 기본 보안 설정 필요
공개 서비스 운영은 홈서버보다 관리형 클라우드가 안정적입니다. 홈서버는 테스트/데모/개인 프로젝트 단계에서 특히 유용합니다.

Cloudflare Tunnel 대안

고정 주소가 필요하고 도메인이 있다면 Cloudflare Tunnel도 좋은 선택지입니다.

cloudflared tunnel create my-tunnel
cloudflared tunnel route dns my-tunnel mysite.com
cloudflared tunnel run my-tunnel

마무리

구형 노트북 + Node.js + 터널링 조합은 비용 대비 학습 효과가 큽니다. 클라우드 비용 없이도 배포, 네트워크, 프로세스 관리 개념을 실전으로 익힐 수 있습니다.

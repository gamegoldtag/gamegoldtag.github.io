Oracle Cloud 무료 서버 제대로 쓰는 법

게시일: 2025년 9월 12일 · 16분 읽기

홈서버는 인터넷/전원 상태에 영향을 크게 받습니다. 24시간 안정적인 구동이 필요하면 클라우드 서버가 훨씬 적합합니다. Oracle Cloud Always Free는 시작 비용을 낮추면서도 꽤 높은 사양을 제공해 개인 프로젝트와 포트폴리오 서버로 많이 활용됩니다.

왜 Oracle Cloud를 선택할까?

Always Free 리소스가 다른 클라우드 대비 상대적으로 넉넉하기 때문입니다. 다만 제공 사양과 정책은 시점별로 변경될 수 있어, 배포 전 콘솔의 현재 한도를 반드시 확인해야 합니다.

Always Free 예시 항목 (정책 변경 가능)
- ARM 기반 VM 리소스 (Ampere A1)
- AMD VM 리소스 (VM.Standard.E2.1.Micro)
- 블록 스토리지 일정량
- 월간 아웃바운드 네트워크 전송량
- 기간 제한 없는 무료 티어 (정책 범위 내)

준비물

  • Oracle Cloud 계정 (cloud.oracle.com 무료 가입)
  • 본인 인증 가능한 신용카드/체크카드 (가입 검증용, 바로 청구되지 않음)
  • SSH 키 쌍 (공개키/개인키)
무료 범위를 넘거나 유료 리소스를 생성하면 과금될 수 있습니다. 인스턴스 생성 화면에서 "Always Free-eligible" 표시를 반드시 확인하세요.

SSH 키 생성

VM에 접속하려면 SSH 키 쌍이 필요합니다. 키 쌍은 공개키(서버에 등록)와 개인키(내 컴퓨터에 보관)로 구성됩니다:

# 터미널에서 실행
ssh-keygen -t rsa -b 4096 -f ~/.ssh/oracle-key

# 생성 확인
ls ~/.ssh/
# oracle-key (개인키), oracle-key.pub (공개키)

공개키 파일(oracle-key.pub)의 내용을 복사해 두면 VM 생성 시 사용합니다.

1단계: Oracle Cloud 계정 생성

cloud.oracle.com에서 계정을 만들고 이메일 인증과 결제 수단 등록을 완료합니다. 계정 승인에 최대 수 시간이 걸릴 수 있습니다.

2단계: VM 인스턴스 생성

Oracle Cloud 콘솔 → Compute → Instances → Create Instance를 선택합니다.

권장 설정
- Name: my-server (원하는 이름)
- Image: Ubuntu 22.04 LTS
- Shape: VM.Standard.A1.Flex 또는 VM.Standard.E2.1.Micro
  (Always Free-eligible 확인 필수)
- Boot Volume: 최소 50GB 또는 기본값

SSH 키 설정에서 앞서 생성한 공개키(oracle-key.pub) 내용을 붙여넣습니다. Create 버튼을 눌러 인스턴스를 생성하고 공인 IP 주소를 메모합니다.

3단계: SSH로 서버 접속

ssh -i ~/.ssh/oracle-key ubuntu@YOUR_PUBLIC_IP

처음 접속 시 fingerprint 확인 메시지가 뜨면 "yes"를 입력합니다. 접속이 되면 서버 초기 업데이트를 진행합니다:

sudo apt update && sudo apt upgrade -y

4단계: Nginx 설치 및 시작

sudo apt install nginx -y
sudo systemctl enable nginx   # 재부팅 시 자동 시작
sudo systemctl start nginx
sudo systemctl status nginx   # 실행 상태 확인

브라우저에서 서버 공인 IP로 접속하면 Nginx 기본 페이지가 표시됩니다. 표시되지 않으면 다음 단계(방화벽 설정)를 진행하세요.

5단계: 방화벽 및 보안 규칙 열기

Oracle Cloud에는 두 개의 방화벽이 있습니다: Oracle 네트워크 보안 그룹(콘솔에서 설정)과 서버 내부 방화벽(UFW). 둘 다 열어야 외부에서 접속 가능합니다.

Oracle 콘솔에서 인그레스 규칙 추가

Networking → Virtual Cloud Networks → VCN → Security Lists → Default Security List에서 다음 규칙을 추가합니다:

HTTP (웹):  TCP 포트 80,  소스 0.0.0.0/0
HTTPS:      TCP 포트 443, 소스 0.0.0.0/0

서버 내부 UFW 설정

sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw enable
sudo ufw status

6단계: 웹사이트 파일 업로드

SCP로 파일 전송

# 로컬 PC에서 실행
scp -i ~/.ssh/oracle-key index.html ubuntu@YOUR_PUBLIC_IP:/tmp/index.html

# 서버에서 실행
ssh -i ~/.ssh/oracle-key ubuntu@YOUR_PUBLIC_IP
sudo mv /tmp/index.html /var/www/html/index.html

Nginx 설정 파일 (선택)

기본 설정으로도 /var/www/html/의 파일이 서빙됩니다. 커스텀 도메인이 있다면 /etc/nginx/sites-available/에 설정 파일을 추가합니다:

sudo nano /etc/nginx/sites-available/mysite

# 내용 입력:
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    root /var/www/html;
    index index.html;
    location / {
        try_files $uri $uri/ =404;
    }
}

sudo ln -s /etc/nginx/sites-available/mysite /etc/nginx/sites-enabled/
sudo nginx -t         # 설정 문법 검사
sudo systemctl reload nginx

7단계: HTTPS 적용 (Let's Encrypt)

도메인이 있고, DNS가 서버 IP를 가리키고 있다면 무료 SSL 인증서를 발급받을 수 있습니다:

sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com

인증서는 90일마다 자동 갱신됩니다. Certbot이 cron 또는 systemd 타이머를 자동으로 설정합니다.

Node.js 앱 배포하기

정적 파일이 아니라 Node.js 백엔드가 필요한 경우:

sudo apt install nodejs npm -y
# 또는 nvm을 사용해 최신 버전 설치

npm install -g pm2

cd ~/myapp
npm install
pm2 start server.js --name myapp
pm2 startup
pm2 save

Nginx를 리버스 프록시로 설정해 80/443 포트의 요청을 Node.js 앱으로 전달합니다:

server {
    listen 80;
    server_name yourdomain.com;
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

운영 시 주의점

  • 장기간 미사용 계정/리소스는 정책에 따라 회수될 수 있습니다. 월 1회 이상 콘솔에 로그인하세요.
  • SSH 키를 분실하면 서버 접속이 불가능해집니다. 개인키를 안전한 곳에 백업하세요.
  • 보안 그룹에서 불필요한 포트는 열어두지 마세요.
  • 주기적으로 sudo apt update && sudo apt upgrade로 보안 업데이트를 적용하세요.
일부 계정에서 장기간 비활성 상태 리소스가 회수될 수 있으므로, 정기 점검 일정을 캘린더에 등록하는 것이 안전합니다.

마무리

Oracle Cloud Always Free는 홈서버보다 훨씬 안정적인 24시간 운영 환경을 저비용으로 시작하기 좋은 선택지입니다. SSH 접속, Nginx 설정, HTTPS 인증서, 방화벽 규칙을 직접 다루면서 서버 관리의 기초를 쌓을 수 있습니다. 핵심은 Always Free 범위를 정확히 지키고, SSH 키와 보안 규칙을 명확히 관리하는 것입니다.

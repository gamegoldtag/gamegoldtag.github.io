돈 없이 내 웹사이트 만드는 법 - AI가 다 해줌

게시일: 2025년 11월 1일 · 13분 읽기

웹사이트 제작은 예전보다 훨씬 저렴해졌고, 초기 단계는 사실상 무료 조합으로 시작할 수 있습니다. 핵심은 "AI로 제작 초안" + "무료 디자인/이미지" + "무료 호스팅" 조합을 현실적으로 쓰는 것입니다. 이 글에서는 각 단계에서 어떤 무료 도구를 쓸 수 있는지, 그리고 언제 유료가 필요해지는지를 정직하게 정리합니다.

항목유료 방식무료 대체

디자인 작업유료 디자인 툴Figma 무료 플랜 / Canva 무료 플랜

단계별 무료 도구 가이드

1. 웹사이트 제작: AI 활용

Claude, ChatGPT 모두 HTML/CSS/JS 초안 제작에 충분합니다. 조건을 구체적으로 전달할수록 결과 품질이 올라갑니다:

요청 예시:
소개 페이지를 만들어주세요.
- 섹션: 자기소개 + 서비스 3개 + 연락처
- 디자인: 흰 배경, 파란색 포인트
- 조건: 모바일 반응형, HTML 한 파일

AI가 만든 초안은 완성품이 아닙니다. 브라우저에서 확인하면서 2~5회 수정 요청을 반복하면 실사용 가능한 수준이 됩니다.

2. 이미지: 라이선스 확인 필수

무료 이미지를 쓸 때 라이선스를 반드시 확인해야 합니다. 상업 사용 가능 여부가 사이트별로 다릅니다:

  • Unsplash: 대부분 무료, 상업·비상업 사용 가능. 저작자 표기 권장
  • Pexels: 무료, 출처 표기 권장. 이미지·영상 모두 제공
  • 직접 제작: Canva 무료 플랜에서 기본 이미지·배너 제작 가능
이미지 라이선스를 확인하지 않고 사용하면 저작권 문제가 생길 수 있습니다. 배포 전 반드시 각 이미지의 라이선스를 확인하세요.

3. 호스팅: 세 가지 무료 선택지

정적 사이트(HTML/CSS/JS만으로 구성)라면 아래 세 서비스 모두 무료로 시작할 수 있습니다:

서비스난이도추천 대상특징
Netlify낮음초보자, 빠른 시작드래그앤드롭 배포
GitHub Pages중간개발자, 코드 이력 관리Git 연동 자동 배포
Cloudflare Pages중간성능 중시, 글로벌 방문자엣지 CDN 배포

무료 서브도메인으로 먼저 시작하기

커스텀 도메인 구입 없이도 각 서비스에서 무료 주소를 발급해 줍니다:

Netlify:          yoursite.netlify.app
GitHub Pages:     yourname.github.io
Cloudflare Pages: yoursite.pages.dev

초기에는 이 주소로도 충분히 공개 테스트가 가능합니다. 사이트 방향이 확정되면 커스텀 도메인을 연결해 브랜드 신뢰도를 높이면 됩니다.

추천 조합 3가지

조합 1: 가장 쉬운 시작 (완전 비개발자)

AI: Claude 또는 ChatGPT (코드 생성)
이미지: Unsplash (무료 고화질)
배포: Netlify (드래그앤드롭)

총 비용: 0원
소요 시간: 30~60분

조합 2: 개발자 친화적

AI: Claude (코드 생성 + 수정)
디자인: Figma 무료 플랜 (레이아웃 시안)
배포: GitHub Pages (Git 기반)

총 비용: 0원
장점: 코드 이력 관리, 협업 가능

조합 3: 성능 중시

AI: ChatGPT 또는 Claude (코드 생성)
이미지: Pexels (무료 스톡)
배포: Cloudflare Pages (엣지 CDN)

총 비용: 0원
장점: 전 세계 빠른 로딩

실제 비용 구조: 무료의 한계

무료 조합이 100% 적용되는 경우와 유료가 필요해지는 경우를 구분하면 계획을 더 정확히 세울 수 있습니다:

사용 목적무료 조합 가능?추가 비용 요소
소개 페이지, 포트폴리오가능커스텀 도메인(선택)
업체 안내, 메뉴판가능커스텀 도메인(권장)
이벤트 랜딩 페이지가능없음
회원가입/로그인 포함어려움백엔드 서비스 필요
결제 기능불가결제 게이트웨이 + 서버
대규모 트래픽한도 초과 가능유료 플랜 전환 검토

커스텀 도메인, 언제 구입할까?

무료 서브도메인으로 시작하되, 아래 상황이 생기면 커스텀 도메인 구입을 고려하세요:

  • 명함이나 SNS 프로필에 사이트 주소를 넣을 때
  • 브랜드 이름을 주소에 반영하고 싶을 때
  • 사이트를 장기적으로 운영할 계획이 생겼을 때

도메인 가격은 서비스에 따라 연간 1~3만 원 수준입니다. 초기 투자 비용으로 부담이 낮은 편입니다.

무료 조합의 현실적인 한계

  • 회원가입/로그인/결제 기능은 추가 백엔드가 필요합니다.
  • 대규모 트래픽에서는 각 서비스의 무료 한도를 확인해야 합니다.
  • AI 생성 코드의 품질 검수(접근성, 모바일 대응, 보안 헤더)가 필요합니다.
  • 무료 서비스 정책은 변경될 수 있으므로 주기적으로 확인이 필요합니다.
"완전 0원"은 정적 소개 사이트 기준에 가깝습니다. 로그인/결제/대규모 트래픽/DB가 필요한 서비스는 별도 비용을 고려해야 합니다.

마무리

핵심은 "무조건 무료"가 아니라 "초기 비용 없이 빠르게 검증"입니다. 아이디어 단계에서는 무료 조합으로 충분히 시작할 수 있고, 성과가 나오면 필요한 부분만 유료로 확장하는 전략이 가장 현실적입니다. 도메인 구입도, 서버 비용도, 처음부터 결정할 필요가 없습니다. 먼저 만들고, 운영해보면서 다음 단계를 결정하면 됩니다.

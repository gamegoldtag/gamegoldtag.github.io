Cloudflare Pages - GitHub Pages보다 빠른 무료 호스팅?

게시일: 2025년 10월 10일 · 14분 읽기

Cloudflare Pages는 전 세계 엣지 네트워크를 기반으로 정적 사이트를 배포할 수 있는 서비스입니다. 무료 플랜만으로도 시작이 가능하고, Direct Upload와 Git 연동을 모두 지원합니다. 특히 방문자가 전 세계에 분산되어 있거나, 한국 이외 지역에서 빠른 로딩이 필요한 경우 눈에 띄는 차이를 경험할 수 있습니다.

Cloudflare Pages가 주목받는 이유

핵심은 엣지(Edge) 배포입니다. 기존 호스팅은 특정 서버 위치에서 파일을 제공하지만, Cloudflare Pages는 전 세계 200여 개 데이터센터에 콘텐츠를 분산해 방문자와 가장 가까운 위치에서 제공합니다.

예를 들어 서울에 있는 서버에서만 파일을 제공하면, 미국·유럽·동남아시아 방문자는 물리적 거리 때문에 로딩이 느립니다. Cloudflare는 이 문제를 자동으로 해결합니다.

트래픽/빌드 횟수/기능 한도는 시점별로 달라질 수 있으므로, 적용 전 Cloudflare 공식 플랜 안내를 확인하세요.

준비물

  • Cloudflare 계정 (cloudflare.com 무료 가입)
  • 배포할 사이트 폴더 (index.html 포함)
  • 선택 사항: GitHub 저장소 (자동 배포용)

배포 방법 2가지

Cloudflare Pages는 두 가지 배포 방식을 모두 지원합니다. 처음 시작할 때는 Direct Upload가 가장 빠르고, 운영 단계로 넘어가면 Git 연동으로 전환하는 것이 효율적입니다.

방식특징추천 상황
Direct Upload폴더 직접 업로드, 즉시 배포처음 시작, 빠른 테스트
Git 연동push 시 자동 빌드/배포운영 단계, 지속 업데이트

방법 1: Direct Upload (초기 배포)

  1. Cloudflare 로그인 후 Workers & Pages로 이동
  2. Create 클릭 후 Pages 탭 선택
  3. Direct Upload를 선택하고 프로젝트 이름 지정
  4. 사이트 파일이 있는 폴더를 업로드 영역에 드래그앤드롭
  5. Deploy site 버튼 클릭

배포 완료 후 project-name.pages.dev 형식의 주소가 발급됩니다.

파일 업데이트 방법

수정 후 재배포하려면 프로젝트 페이지에서 "Create new deployment"를 선택하고 새 폴더를 업로드합니다. 이전 배포 이력도 유지됩니다.

방법 2: GitHub 연동 (자동 배포)

  1. Workers & Pages → Create → Pages → Connect to Git 선택
  2. GitHub 계정 연결 후 저장소 선택
  3. 빌드 설정 입력:
    • 정적 HTML: 빌드 명령 없음, 출력 디렉터리 비워두기
    • React (Vite): 빌드 명령 npm run build, 출력 dist
    • Next.js: 빌드 명령 npm run build, 출력 .next
  4. Save and Deploy 클릭

이후 GitHub에 push할 때마다 자동으로 빌드하고 배포합니다. 배포 상태는 Pages 대시보드에서 실시간으로 확인할 수 있습니다.

커스텀 도메인 연결하기

Cloudflare Pages에서 커스텀 도메인을 연결하면 추가 SSL 설정 없이 자동으로 HTTPS가 적용됩니다. 특히 도메인이 이미 Cloudflare에서 관리되고 있다면 클릭 몇 번으로 연결됩니다.

도메인 연결 과정

  1. Pages 프로젝트 → Custom domains → Add a custom domain
  2. 도메인 입력 후 Continue
  3. Cloudflare 관리 도메인: 자동으로 DNS 레코드 추가
  4. 외부 도메인: CNAME 레코드를 도메인 관리 패널에서 직접 추가
CNAME 레코드 예시:
이름: www (또는 @)
값: project-name.pages.dev

Cloudflare Pages 선택이 유리한 상황

  • 사이트 방문자가 한국 이외 지역에도 분산되어 있는 경우
  • 이미 Cloudflare에서 도메인을 관리하고 있는 경우
  • 가능한 빠른 로딩 속도가 중요한 경우 (이커머스, 랜딩 페이지)
  • Direct Upload와 Git 연동을 상황에 따라 유연하게 전환하고 싶은 경우

자주 막히는 상황

  • 배포 후 "Page Not Found" 오류: index.html이 폴더 루트에 있는지 확인
  • 이미지 안 보임: 이미지 파일이 같은 폴더에 있고, 파일명 대소문자가 정확한지 확인
  • React/Vue 빌드 실패: Node.js 버전을 프로젝트 요건에 맞게 Environment Variables에서 설정
  • 커스텀 도메인 SSL 오류: DNS 설정 후 최대 24시간까지 대기 필요

무엇을 선택할까?

  • 완전 입문, 빠른 시작: Netlify
  • Git 기반 개발 워크플로: GitHub Pages
  • 글로벌 성능, 확장성: Cloudflare Pages
  • Next.js 프로젝트: Vercel

마무리

Cloudflare Pages는 무료 시작, 빠른 배포, Git 자동화까지 균형이 좋은 선택지입니다. 특히 방문자 지역이 다양한 사이트나, 이미 Cloudflare로 도메인을 관리하는 경우라면 후보군에 반드시 넣을 만합니다. 처음 배포는 Direct Upload로 빠르게 시작하고, 운영이 안정화되면 Git 연동으로 전환하는 방식이 가장 효율적입니다.

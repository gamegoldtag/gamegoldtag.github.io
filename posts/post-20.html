Vercel/Netlify로 프론트엔드 무료 배포하기

게시일: 2025년 10월 3일 · 15분 읽기

React, Vue, Next.js 프로젝트를 배포할 때 Vercel과 Netlify는 가장 많이 쓰이는 선택지입니다. 두 서비스 모두 Git 기반 자동 배포를 제공하고, 무료 플랜으로 시작하기 좋습니다. 이 글에서는 각 서비스의 배포 과정, 환경 변수 관리, 프리뷰 배포, 커스텀 도메인 설정까지 실무에서 필요한 내용을 정리합니다.

왜 많이 쓰일까?

핵심은 빌드 자동화입니다. 이전에는 빌드를 로컬에서 직접 실행하고 결과물을 FTP나 서버에 수동으로 올려야 했습니다. 두 서비스를 사용하면 이 과정이 완전히 자동화됩니다:

기존 방식:
로컬 npm run build → dist 폴더 생성 → FTP 업로드 → 캐시 갱신

자동화 후:
git push → 플랫폼이 빌드 → 자동 배포 → 즉시 반영

Netlify 배포 상세 가이드

Netlify는 입문자가 시작하기 쉬운 UI를 제공합니다. 특히 정적 사이트는 드래그앤드롭 배포가 간단하고, 프레임워크 프로젝트도 몇 번의 클릭으로 설정이 완료됩니다.

방법 1: 드래그앤드롭 (비개발자 추천)

  1. 로컬에서 npm run build 실행
  2. Netlify 로그인 후 대시보드로 이동
  3. 빌드 결과 폴더(dist 또는 build)를 드래그앤드롭
  4. 배포 완료 후 URL 확인

방법 2: GitHub 연동 (운영 단계 추천)

  1. Add new site → Import from Git
  2. GitHub 저장소 선택
  3. 빌드 명령과 출력 디렉터리 입력:
    • React (Vite): 빌드 npm run build, 출력 dist
    • Create React App: 빌드 npm run build, 출력 build
    • Vue 3: 빌드 npm run build, 출력 dist
  4. Deploy site 클릭

Netlify 환경 변수 설정

API 키 같은 민감한 설정은 Site settings → Environment variables에서 관리합니다:

예시:
VITE_API_BASE_URL = https://api.example.com
VITE_GOOGLE_ANALYTICS = G-XXXXXXXXXX

Vercel 배포 상세 가이드

Vercel은 프레임워크 자동 감지와 기본 설정이 강점입니다. 특히 Next.js는 Vercel이 직접 만든 프레임워크라 서버 컴포넌트, API 라우트, ISR 등 고급 기능이 가장 잘 동작합니다.

  1. vercel.com 로그인 후 Add New → Project
  2. GitHub 저장소 연결 및 저장소 선택
  3. 프레임워크 자동 감지 확인 (Next.js, React, Vue 등 자동 인식)
  4. 환경 변수가 있으면 Environment Variables 탭에서 추가
  5. Deploy 클릭

Vercel 환경 변수 설정

Project Settings → Environment Variables
변수명: NEXT_PUBLIC_API_URL
값: https://api.example.com
범위: Production / Preview / Development (분리 설정 가능)

프리뷰 배포 활용하기

두 서비스 모두 PR(Pull Request)이나 브랜치 push 시 자동으로 프리뷰 URL을 생성합니다. 이 기능은 팀 협업에서 특히 유용합니다:

  • feature 브랜치를 push하면 자동으로 임시 URL이 생성됨
  • 해당 URL을 팀원과 공유해 기능 확인 가능
  • main에 merge되면 자동으로 프로덕션에 반영
예시 워크플로:
feature/new-header 브랜치 push
→ https://project-git-feature-new-header.netlify.app 생성
→ 팀원이 URL로 확인 후 승인
→ main에 merge
→ 프로덕션 자동 배포

커스텀 도메인 연결

두 서비스 모두 무료 플랜에서 커스텀 도메인 연결이 가능합니다.

Netlify 도메인 연결

  1. Site settings → Domain management → Add custom domain
  2. 도메인 입력 후 Verify
  3. 도메인 관리 패널에서 DNS 레코드 추가:
    • CNAME: www → project.netlify.app
    • A 레코드: @ → Netlify의 공인 IP
  4. SSL 인증서 자동 발급 (Let's Encrypt)

Vercel 도메인 연결

  1. Project Settings → Domains → Add
  2. 도메인 입력 후 Add
  3. 제시된 DNS 레코드를 도메인 관리 패널에 추가
  4. 자동 SSL 적용 확인

프레임워크별 선택 가이드

  • Next.js: Vercel 우선 검토 (공식 지원, 서버 컴포넌트/ISR 최적화)
  • React (Vite/CRA): Netlify/Vercel 모두 적합
  • Vue 3: Netlify/Vercel 모두 적합
  • Svelte/Astro: 두 서비스 모두 지원, 빌드 설정 확인 필요
  • 정적 HTML: Netlify 드래그앤드롭이 가장 간단

자주 막히는 문제

  • 빌드 실패 (Module not found): package.json의 의존성이 모두 설치되어 있는지 확인. npm install 후 재시도
  • 환경 변수 읽기 실패: 변수명이 정확한지, Vite는 VITE_, CRA는 REACT_APP_ 접두사 확인
  • SPA 새로고침 404: Netlify에서는 _redirects 파일에 /* /index.html 200 추가
  • 빌드 시간이 너무 오래 걸림: node_modules가 캐시되지 않는 경우. 캐시 설정 확인

마무리

두 서비스 모두 무료 시작이 가능하고 실무에서도 충분히 사용됩니다. 처음에는 설정이 쉬운 쪽으로 시작하고, 프로젝트 특성(프레임워크/팀 워크플로)에 맞춰 고르는 것이 가장 효율적입니다. Next.js 프로젝트는 Vercel, 그 외 React/Vue 프로젝트는 Netlify로 시작하면 대부분의 상황에서 무난하게 운영됩니다.

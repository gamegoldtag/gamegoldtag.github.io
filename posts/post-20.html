Vercel/Netlify로 프론트엔드 무료 배포하기

게시일: 2025년 10월 3일 · 12분 읽기

React, Vue, Next.js 프로젝트를 배포할 때 Vercel과 Netlify는 가장 많이 쓰이는 선택지입니다. 두 서비스 모두 Git 기반 자동 배포를 제공하고, 무료 플랜으로 시작하기 좋습니다.

왜 많이 쓰일까?

핵심은 빌드 자동화입니다. 소스코드를 push하면 빌드와 배포를 플랫폼이 대신 처리합니다.

로컬 빌드 + 수동 업로드
→ Git push + 자동 빌드/배포

Netlify 배포

Netlify는 입문자가 시작하기 쉬운 UI를 제공합니다. 특히 정적 사이트는 드래그앤드롭 배포가 간단합니다.

방법 1: 드래그앤드롭

  1. npm run build 실행
  2. Netlify 로그인
  3. 빌드 결과 폴더(dist 등) 업로드

방법 2: GitHub 연동

  1. Add new site → Import from Git
  2. 저장소 선택
  3. 빌드 명령/출력 경로 입력
  4. Deploy 실행

Vercel 배포

Vercel은 프레임워크 자동 감지와 기본 설정이 강점입니다. Next.js 프로젝트에서 특히 운영 경험이 좋습니다.

  1. Vercel 로그인 후 새 프로젝트 생성
  2. GitHub 저장소 연결
  3. 프레임워크 감지 확인
  4. Deploy 실행

비교 포인트

자동 배포지원지원
트래픽/빌드 시간/무료 한도는 플랜 변경 가능성이 있으므로, 실제 배포 전에 각 서비스 공식 요금표를 확인하세요.

프레임워크별 선택 가이드

  • Next.js: Vercel 우선 검토
  • React/Vue/Svelte: Netlify/Vercel 모두 적합
  • 정적 HTML: Netlify의 업로드 배포가 간단

배포 자동화 흐름

코드 수정 → git push → 자동 빌드 → 자동 배포 → 사이트 반영

이 흐름을 만들면 수동 업로드, FTP, 서버 관리 부담이 크게 줄어듭니다.

마무리

두 서비스 모두 무료 시작이 가능하고 실무에서도 충분히 사용됩니다. 처음에는 설정이 쉬운 쪽으로 시작하고, 프로젝트 특성(프레임워크/팀 워크플로)에 맞춰 고르는 것이 가장 효율적입니다.

GitHub Pages로 내 사이트 10분만에 올리기

게시일: 2025년 10월 18일 · 14분 읽기

GitHub Pages는 정적 사이트를 무료로 공개하기 좋은 호스팅입니다. 포트폴리오, 프로젝트 데모, 문서 사이트에 특히 잘 맞고, Git 기반 워크플로를 쓰는 사람에게 유지보수가 편합니다. 이 글에서는 처음 사용하는 분도 따라할 수 있도록 전 과정을 상세히 정리합니다.

GitHub Pages란?

GitHub 저장소에 올린 정적 파일(HTML/CSS/JS)을 웹사이트로 배포해주는 무료 서비스입니다. HTTPS가 기본 제공되고, 기본 서브도메인(username.github.io)과 커스텀 도메인 연결 모두 가능합니다.

GitHub Pages가 특히 강한 상황:

  • 개발자 포트폴리오 (GitHub 계정과 연결이 자연스러움)
  • 오픈소스 프로젝트 문서 페이지
  • 코드 변경 이력을 남기며 운영하고 싶은 사이트
  • 팀 프로젝트에서 여러 명이 함께 관리하는 사이트
트래픽·용량·빌드 제한은 시점별로 변경될 수 있으니, 배포 전 GitHub 공식 문서를 확인하는 것이 안전합니다.

준비물

  • GitHub 계정 (github.com 무료 가입)
  • index.html 중심의 사이트 파일

1단계: GitHub 가입 및 사용자명 선택

github.com에서 가입합니다. 사용자명(username)은 기본 사이트 주소와 직접 연결되므로, 포트폴리오 용도라면 실명이나 닉네임을 신중하게 선택하세요.

username: honggildong
기본 주소: https://honggildong.github.io

2단계: 저장소 생성

GitHub 화면 우측 상단의 + 버튼 → New repository를 선택합니다.

저장소 이름 규칙

  • 개인 루트 사이트: 저장소 이름을 username.github.io 형식으로 만들면 https://username.github.io로 접속됩니다.
  • 프로젝트 사이트: my-project처럼 일반 이름을 사용하면 https://username.github.io/my-project로 접속됩니다.

공개(Public) 저장소로 생성해야 무료로 GitHub Pages를 사용할 수 있습니다.

3단계: 파일 업로드

저장소 페이지에서 "Add file" → "Upload files"를 선택합니다.

  • index.html을 업로드하고 커밋합니다.
  • 이미지가 있으면 이미지 파일도 같은 경로에 함께 업로드해야 화면이 깨지지 않습니다.
  • 파일 여러 개를 한 번에 드래그앤드롭으로 업로드할 수 있습니다.
업로드 후 커밋 메시지 예시:
Add portfolio site files

4단계: GitHub Pages 활성화

저장소 상단 메뉴에서 Settings → Pages로 이동합니다.

  1. Source 항목에서 "Deploy from a branch" 선택
  2. Branch를 "main", 폴더를 "/ (root)"로 설정
  3. Save 클릭

설정 후 1~5분 내로 사이트 URL이 활성화됩니다. Pages 설정 화면에서 URL을 확인할 수 있습니다.

5단계: 수정 및 업데이트

파일을 수정해 커밋하면 자동으로 재배포됩니다. 수정 방법은 두 가지입니다:

방법 1: 웹에서 직접 수정

저장소에서 파일을 클릭 → 연필 아이콘(Edit) 선택 → 내용 수정 → 커밋. 간단한 텍스트 수정에 적합합니다.

방법 2: 로컬에서 수정 후 업로드

파일을 로컬에서 수정한 뒤, "Upload files"로 다시 업로드합니다. 이미지 변경이나 대규모 수정에 적합합니다.

커스텀 도메인 연결하기

도메인을 구입했다면 GitHub Pages에 연결해 yourname.github.io 대신 yourname.com으로 접속되게 할 수 있습니다.

CNAME 파일 추가

저장소 루트에 CNAME 파일을 만들고 도메인을 입력합니다:

yourdomain.com

도메인 DNS 설정

도메인 구입처(가비아, 후이즈 등)에서 DNS 설정에 GitHub Pages IP를 추가합니다:

A 레코드:
185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153

또는 CNAME 레코드:
username.github.io

DNS 전파에는 최대 24~48시간이 걸릴 수 있습니다.

React/Vue 프로젝트 GitHub Actions 자동화

정적 HTML이 아니라 React, Vue 같은 프레임워크 프로젝트라면 빌드 결과물을 배포해야 합니다. GitHub Actions로 자동화하면 push만으로 배포 가능합니다:

# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist

이 워크플로를 추가하면 main 브랜치에 push할 때마다 자동으로 빌드하고 배포합니다.

자주 막히는 상황

  • Pages 설정 후 URL이 활성화 안 됨: 5~10분 기다린 후 페이지 새로고침
  • 저장소를 Private으로 만들면 Pages 안 됨: Settings에서 Public으로 변경
  • 이미지 깨짐: 이미지 파일이 저장소에 있는지, 경로가 대소문자까지 정확한지 확인
  • 커스텀 도메인 HTTPS 오류: DNS 전파 완료 후 Settings → Pages에서 "Enforce HTTPS" 체크

마무리

GitHub Pages는 한 번 설정하면 "수정 → 커밋 → 자동 반영" 흐름이 안정적으로 돌아갑니다. 특히 개발자라면 코드 작업과 배포가 같은 도구(GitHub)에서 이루어지기 때문에 관리 부담이 크게 줄어듭니다. 코드 이력을 남기면서 포트폴리오나 문서형 사이트를 장기적으로 운영하려는 분에게 가장 적합한 선택입니다.

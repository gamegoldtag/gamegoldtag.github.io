GitHub Pages로 내 사이트 10분만에 올리기

게시일: 2025년 10월 18일 · 11분 읽기

GitHub Pages는 정적 사이트를 무료로 공개하기 좋은 호스팅입니다. 포트폴리오, 프로젝트 데모, 문서 사이트에 특히 잘 맞고, Git 기반 워크플로를 쓰는 사람에게 유지보수가 편합니다.

GitHub Pages란?

GitHub 저장소에 올린 정적 파일(HTML/CSS/JS)을 웹사이트로 배포해주는 서비스입니다. HTTPS와 기본 서브도메인을 제공하며 커스텀 도메인 연결도 가능합니다.

트래픽/용량/빌드 제한은 시점별로 변경될 수 있으니, 배포 전 GitHub 공식 문서를 확인하는 것이 안전합니다.

준비물

  • GitHub 계정
  • index.html 중심의 사이트 파일

1단계: GitHub 가입

github.com에서 가입합니다. 사용자명(username)은 사이트 주소와 연결되므로 신중하게 정하는 편이 좋습니다.

username: honggildong
주소: https://honggildong.github.io

2단계: 저장소 생성

New repository에서 저장소 이름을 username.github.io 형식으로 만듭니다. 예: honggildong.github.io

공개(Public) 저장소로 생성합니다.

3단계: 파일 업로드

저장소 페이지에서 기존 파일 업로드 메뉴를 선택하고 index.html을 올린 뒤 커밋합니다. 이미지가 있으면 같은 경로에 함께 업로드해야 화면이 깨지지 않습니다.

4단계: Pages 활성화

Settings → Pages로 이동해 배포 브랜치(main)를 지정하고 저장합니다. 몇 분 내로 사이트 URL이 활성화됩니다.

5단계: 수정 반영

파일을 수정해 커밋하면 자동으로 재배포됩니다. 자주 업데이트한다면 GitHub Desktop 또는 CLI로 로컬-원격 동기화 흐름을 만드는 것이 편합니다.

React/Vue 프로젝트 배포 자동화

정적 HTML이 아니라 프레임워크 프로젝트라면 빌드 결과물을 배포해야 합니다. GitHub Actions로 자동화하면 push만으로 배포 가능합니다.

name: Deploy
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci && npm run build
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist

마무리

GitHub Pages는 한 번 설정하면 "수정 → 커밋 → 자동 반영" 흐름이 안정적으로 돌아갑니다. 코드 이력을 남기면서 운영하고 싶은 포트폴리오/문서형 사이트에 특히 적합합니다.

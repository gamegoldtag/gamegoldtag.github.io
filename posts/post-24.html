상황별 무료 호스팅 선택 가이드 - 뭘 써야 하나?

게시일: 2025년 9월 5일 · 12분 읽기

무료 호스팅은 선택지가 많아 처음엔 오히려 결정이 어렵습니다. 이 글은 지금까지 다룬 서비스들을 목적 중심으로 정리해, "지금 내 상황에 맞는 한 가지"를 빠르게 고를 수 있게 만든 체크 가이드입니다.

한눈에 보는 비교

서비스난이도백엔드상시운영추천 대상
Netlify낮음제한적가능입문자, 첫 배포
Cloudflare Pages낮음제한적가능성능/글로벌 트래픽
GitHub Pages중간불가(정적 중심)가능개발자 포트폴리오
Vercel낮음프레임워크 기반가능Next.js/프론트엔드
Oracle Cloud높음가능가능백엔드 포함 서비스
구형 노트북중간가능조건부테스트/개인 운영
안드로이드 폰중간가능조건부임시 데모
트래픽/빌드 시간/무료 한도는 서비스 정책에 따라 바뀔 수 있습니다. 실제 배포 전 공식 요금/제한 문서를 확인하세요.

상황별 추천

  • 소개 페이지 하나: Netlify
  • 개발자 포트폴리오: GitHub Pages 또는 Netlify
  • React/Vue 프로젝트: Netlify 또는 Vercel
  • Next.js 프로젝트: Vercel 우선
  • 글로벌 트래픽/엣지 성능: Cloudflare Pages
  • 백엔드 API 필요: Oracle Cloud
  • 빠른 내부 데모: 구형 노트북 + ngrok
  • 즉석 임시 공유: 안드로이드 폰 + Termux/ngrok

목적별 조합 예시

완전 초보자

AI로 index.html 생성 → Netlify 드래그앤드롭 배포

개발자 포트폴리오

GitHub 저장소 → GitHub Pages 자동 배포

풀스택 개인 프로젝트

프론트: Vercel/Netlify
백엔드: Oracle Cloud
DB: 관리형 무료 플랜(Supabase 등)

처음 시작할 때 추천 순서

1) Netlify로 먼저 공개
2) 수정 주기가 늘면 Git 연동 자동배포
3) 백엔드가 필요해지면 Oracle Cloud 추가

대부분의 개인 사이트는 1단계만으로도 충분히 운영됩니다.

마무리

정답은 "최고의 서비스"가 아니라 "현재 목적에 맞는 서비스"입니다. 무료 호스팅은 갈아타기 비용이 낮으니, 가장 쉬운 선택으로 먼저 시작하고 필요할 때 확장하는 방식이 가장 실용적입니다.

상황별 무료 호스팅 선택 가이드 - 뭘 써야 하나?

게시일: 2025년 9월 5일 · 15분 읽기

무료 호스팅은 선택지가 많아 처음엔 오히려 결정이 어렵습니다. Netlify, GitHub Pages, Cloudflare Pages, Vercel, Oracle Cloud… 각 서비스마다 특징이 다르고 장단점이 있습니다. 이 글은 지금까지 다룬 서비스들을 목적 중심으로 정리해, "지금 내 상황에 맞는 한 가지"를 빠르게 고를 수 있게 만든 체크 가이드입니다.

한눈에 보는 비교

각 서비스의 핵심 특징

Netlify - 입문자의 첫 선택

드래그앤드롭 배포로 기술적 배경 없이 5분 만에 배포를 완료할 수 있습니다. HTML 한 파일부터 React 프로젝트까지 모두 지원하고, 서버리스 함수(Netlify Functions)로 간단한 백엔드 기능도 추가할 수 있습니다.

  • 강점: 가장 쉬운 시작, 빠른 배포
  • 약점: 글로벌 CDN 성능은 Cloudflare보다 다소 낮음

GitHub Pages - 개발자 워크플로의 표준

Git 저장소와 직접 연결되어 코드 변경 이력과 배포가 한 곳에서 관리됩니다. 개발자라면 이미 GitHub를 사용하고 있어 별도 서비스 가입 없이 시작할 수 있습니다. GitHub Actions로 프레임워크 빌드도 자동화할 수 있습니다.

  • 강점: 코드 이력 통합 관리, 오픈소스 프로젝트 페이지
  • 약점: 동적 기능 없음, Git 기초 지식 필요

Cloudflare Pages - 글로벌 성능 우선

전 세계 200여 개 엣지 데이터센터에 분산 배포하여 어디서든 빠른 로딩이 가능합니다. Netlify와 비슷하게 Direct Upload와 Git 연동을 모두 지원합니다.

  • 강점: 최고 수준의 글로벌 CDN 성능
  • 약점: 초보자에게는 설정이 다소 복잡할 수 있음

Vercel - Next.js의 공식 플랫폼

Next.js를 만든 팀이 운영하는 플랫폼으로, SSR·ISR·서버 컴포넌트 같은 고급 기능이 가장 잘 동작합니다. React, Vue, Svelte도 지원하고, 프리뷰 배포와 환경 변수 관리가 편리합니다.

  • 강점: Next.js 최적화, 프리뷰 배포, 서버리스 함수
  • 약점: 팀 플랜의 가격이 높은 편

Oracle Cloud Always Free - 백엔드가 필요한 경우

실제 리눅스 VM을 무료로 제공해 Node.js, Python, Java 등 어떤 백엔드 스택도 실행할 수 있습니다. 데이터베이스, 파일 서버, API 서버 등 복잡한 구성이 가능합니다.

  • 강점: 완전한 서버 환경, 백엔드 자유도
  • 약점: 리눅스/SSH 지식 필요, 초기 설정 복잡

상황별 추천

  • 소개 페이지 하나: Netlify (드래그앤드롭, 5분 완료)
  • 개발자 포트폴리오: GitHub Pages (코드 이력 + 포트폴리오 통합)
  • React/Vue 프로젝트: Netlify 또는 Vercel
  • Next.js 프로젝트: Vercel 우선 (공식 지원)
  • 글로벌 트래픽/엣지 성능: Cloudflare Pages
  • 백엔드 API + DB 필요: Oracle Cloud
  • 빠른 내부 데모: 구형 노트북 + ngrok
  • 즉석 임시 공유: 안드로이드 폰 + Termux/ngrok

목적별 조합 예시

완전 초보자 (코딩 경험 없음)

1단계: AI(Claude/ChatGPT)로 index.html 생성
2단계: Netlify 드래그앤드롭 배포
3단계: *.netlify.app 주소 공유

개발자 포트폴리오

1단계: GitHub 저장소 생성
2단계: index.html 또는 React 프로젝트 업로드
3단계: GitHub Pages 활성화
4단계: username.github.io 주소 공유

풀스택 개인 프로젝트

프론트엔드: Vercel 또는 Netlify (React/Next.js)
백엔드 API: Oracle Cloud VM (Node.js/Python)
데이터베이스: Supabase 무료 플랜 (PostgreSQL)
스토리지: Cloudflare R2 무료 티어

팀 프로젝트 프리뷰 배포

메인 배포: Vercel (자동 프리뷰 URL 생성)
PR 생성 → 자동 프리뷰 URL 발급
팀원 확인 → 승인 → main merge → 자동 프로덕션 배포

처음 시작할 때 추천 순서

1단계: Netlify로 먼저 공개 (가장 빠름)
2단계: 수정 주기가 늘면 Git 연동 자동배포로 전환
3단계: 백엔드가 필요해지면 Oracle Cloud 추가
4단계: 트래픽이 늘면 Cloudflare Pages로 이전 검토

대부분의 개인 사이트는 1단계만으로도 충분히 운영됩니다. 단계를 건너뛰지 말고 현재 필요에만 집중하세요.

호스팅 갈아타기 비용은?

무료 호스팅 서비스들의 갈아타기 비용은 매우 낮습니다. 정적 사이트라면 파일을 새 서비스에 업로드하면 끝입니다. 커스텀 도메인이 있다면 DNS 레코드를 새 서비스로 변경하면 됩니다.

이것이 무료 호스팅의 중요한 장점입니다. 처음 선택이 완벽하지 않아도 쉽게 이동할 수 있습니다. 가장 쉬운 선택으로 먼저 시작하고, 더 좋은 선택이 필요해질 때 이동하면 됩니다.

마무리

정답은 "최고의 서비스"가 아니라 "현재 목적에 맞는 서비스"입니다. 무료 호스팅은 갈아타기 비용이 낮으니, 가장 쉬운 선택으로 먼저 시작하고 필요할 때 확장하는 방식이 가장 실용적입니다. 비개발자라면 Netlify로, 개발자라면 GitHub Pages나 Vercel로 시작하는 것을 권장합니다.

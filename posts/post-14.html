코딩 몰라도 AI로 내 포트폴리오 사이트 만들기

게시일: 2025년 11월 15일 · 14분 읽기

취업 준비나 프리랜서 활동을 하다 보면 포트폴리오 사이트가 필요해집니다. "개발을 배워야 하나?"라는 고민이 생기지만, AI 도구를 활용하면 코딩 경험 없이도 기본형 사이트를 만들 수 있습니다. 핵심은 코드 작성 자체보다, 원하는 구조를 정확히 설명하고 반복 수정하는 과정입니다.

이 글은 디자이너, 사진작가, 프리랜서, 취준생 등 코딩 경험이 없는 분들을 위한 실전 가이드입니다.

포트폴리오 사이트에 꼭 들어가야 하는 것들

AI에게 요청하기 전에 포함할 내용을 먼저 정해두면 결과물 품질이 크게 올라갑니다. 기본 포트폴리오에는 다음 네 섹션이 필수입니다:

  • 자기소개: 이름, 직업/역할, 한 줄 슬로건 또는 강점 요약
  • 전문 분야 또는 기술 스택: 무엇을 잘하는지 구체적으로
  • 작업물/프로젝트: 대표 작업 3~6개, 각 작업 제목+간단 설명
  • 연락 채널: 이메일, SNS, 오픈 카톡 등 실제 연락 가능한 수단

1단계: 내 정보 먼저 정리하기

프롬프트를 작성하기 전에 아래 형식으로 정보를 먼저 정리하세요. 이 내용이 AI 요청의 기반이 됩니다.

이름: 홍길동
직업: 프리랜서 그래픽 디자이너
한 줄 소개: 브랜드의 첫인상을 만드는 디자이너
전문 분야: 로고 디자인, 브랜딩, SNS 콘텐츠 제작
대표 작업물:
  - A카페 로고 리브랜딩 (2024)
  - B쇼핑몰 SNS 콘텐츠 패키지 (2024)
  - C스타트업 명함/인쇄물 디자인 (2023)

2단계: AI에게 구체적으로 요청하기

ChatGPT 또는 Claude에 기능과 디자인 조건을 함께 지정해서 요청합니다. 막연한 요청보다 레이아웃, 색상, 섹션 구성, 기술 요건을 명시할수록 수정 횟수가 줄어듭니다.

실전 프롬프트 예시 (그래픽 디자이너)

포트폴리오 웹사이트를 HTML로 만들어주세요.

[내 정보]
이름: 홍길동
직업: 프리랜서 그래픽 디자이너
소개: 브랜드의 첫인상을 만드는 디자이너

[섹션 구성]
1) 상단 히어로: 이름 + 직업 + 짧은 소개문 + 연락하기 버튼
2) 전문 분야: 카드 3개 (로고 디자인 / 브랜딩 / SNS 콘텐츠)
3) 대표 작업물: 카드 3개, 각각 이미지 영역 + 제목 + 설명 1줄
4) 연락처: 이메일 아이콘 + 인스타그램 링크

[디자인 조건]
- 배경: 흰색, 포인트 컬러: 남색(#1a237e)
- 폰트: Pretendard (CDN)
- 모바일 반응형 필수
- HTML/CSS/JS 한 파일

[기타]
- 한국어
- 이미지는 work1.jpg, work2.jpg, work3.jpg 파일명 사용

직업별 프롬프트 포인트

  • 개발자: 기술 스택 배지, GitHub 링크, 프로젝트 데모 버튼 요청
  • 사진작가: 풀스크린 그리드 갤러리, 라이트박스 효과 요청
  • 번역가/작가: 작업 유형 목록, 고객사 로고, 후기 섹션 요청
  • 강사/컨설턴트: 경력 타임라인, 수강 후기, 강의 목록 요청

3단계: 코드 저장 후 브라우저에서 확인

AI가 생성한 코드를 복사해 index.html로 저장하고 브라우저에서 직접 확인합니다.

  • Windows: 메모장 실행 → 코드 붙여넣기 → "파일 형식: 모든 파일"로 변경 → index.html로 저장
  • macOS: TextEdit 실행 → 포맷 메뉴 → "일반 텍스트 만들기" → 붙여넣기 → index.html로 저장
  • VS Code 추천: 설치 후 New File → index.html 저장 → 코드 붙여넣기 (미리보기 기능 활용 가능)

4단계: 작은 단위로 수정 반복하기

초안을 확인한 뒤, 마음에 안 드는 부분을 한 번에 2~3개씩 묶어서 수정 요청합니다. 한꺼번에 너무 많이 바꾸면 코드가 뒤섞일 수 있습니다.

아래 3가지만 수정해주세요:
1) 작업물 카드의 이미지 영역 높이를 더 크게 (현재 너무 작음)
2) 연락하기 버튼에 hover 시 색상 변화 효과 추가
3) 모바일에서 카드가 1열로 표시되도록 수정

5단계: 실제 작업물 이미지 교체하기

AI 초안에는 대체 이미지 URL이 들어가는 경우가 많습니다. 실제 작업물 이미지를 사용하려면:

  1. 작업물 이미지를 index.html과 같은 폴더에 넣기
  2. 파일명을 코드에 지정한 이름과 동일하게 맞추기
  3. 이미지 크기가 너무 크면 최대 너비 1200px 이하로 줄이기 (로딩 속도 개선)
<!-- 코드 안의 이미지 태그 -->
<img src="work1.jpg" alt="A카페 로고 리브랜딩">
<img src="work2.jpg" alt="B쇼핑몰 SNS 콘텐츠">
<img src="work3.jpg" alt="C스타트업 인쇄물">
작업물 이미지가 없으면 Unsplash(unsplash.com)에서 관련 이미지를 라이선스 확인 후 임시로 사용할 수 있습니다.

6단계: 무료 호스팅으로 배포하기

Netlify는 폴더를 드래그앤드롭하면 즉시 배포되어 입문자에게 가장 쉽습니다.

Netlify 배포 순서

  1. netlify.com 가입 (이메일 또는 GitHub 계정 사용)
  2. 대시보드에서 "Add new site" → "Deploy manually" 클릭
  3. index.html과 이미지 파일이 있는 폴더를 화면에 드래그앤드롭
  4. 업로드 완료 후 *.netlify.app 주소 확인
  5. 주소를 SNS, 이메일 서명, 명함 등에 활용

완성도를 높이는 체크리스트

공개 전 아래 항목을 확인하면 완성도가 크게 올라갑니다:

  • 데스크톱과 스마트폰에서 모두 레이아웃 확인
  • 이름, 연락처, 이메일 주소 오기입 여부 확인
  • 이미지 alt 텍스트가 실제 내용을 설명하는지 확인
  • 페이지 제목(<title>)이 이름 또는 사이트 목적을 담고 있는지 확인
  • 외부 링크(SNS, GitHub 등)가 실제로 연결되는지 확인

한계와 체크포인트

  • 복잡한 기능(로그인/결제/예약 시스템)은 별도 백엔드가 필요합니다.
  • AI 코드 품질은 요청 문맥에 따라 달라집니다. 결과가 이상하면 요청을 더 구체적으로 다듬으세요.
  • 저작권, 개인정보 노출, 연락처 오기입을 배포 전에 반드시 확인하세요.
무료 플랜의 기능과 한도는 시점별로 변경될 수 있으므로, 배포 전에 각 서비스 공식 안내를 확인하는 것을 권장합니다.

마무리

정리하면 "내 정보 정리 → AI 요청 → 로컬 확인 → 수정 반복 → 이미지 교체 → 배포" 순서입니다. 코드를 완벽히 이해하지 못해도, 명확한 요구와 검수 습관만 있으면 실사용 가능한 포트폴리오 사이트를 만들 수 있습니다. 첫 버전이 완벽할 필요는 없습니다. 공개 후 피드백을 받으며 점진적으로 개선해나가는 것이 가장 현실적인 방식입니다.
